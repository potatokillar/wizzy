# Project Instructions

## Scope

- Application code, tests, build config, and runtime assets live under `app/`.
- Design and planning documents live under `docs/`.

## Documentation

- Any behavior, architecture, runtime flow, risk rule, persistence shape, or TUI interaction change must update the relevant docs in the same change.

## Code Changes

- When modifying or adding code, include concise comments for non-trivial logic and important invariants.
- Prefer comments that explain why a rule exists or what must remain true.
- Do not add obvious line-by-line comments.
